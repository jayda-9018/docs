title: "Predict Duration"
description: "音声を生成せずに予想される長さのみを返すAPI、課金予測やテキスト長の調整に活用できます。"
openapi: "/openapi.json POST /sandbox/predict-duration/{voice_id}"
---

このAPIは**実際に音声を生成せずに**、\
入力されたテキストに基づいて予想される音声の長さ（秒単位）のみを返します。
TTS呼び出し前に**予想クレジット消費量を把握したり、テキストの長さを調整したりする際に役立ち**ます。

### 使用方法
- 呼び出し方法とRequest Bodyは`text-to-speech` APIとほぼ同じです。
- ただし、結果として音声ではなく`duration`値のみが返されます。
- Predict Duration API呼び出し時には**クレジットが消費されません。**

### Request Body項目説明
| 項目               | 必須 | 説明                                     |
| :--------------- | :-- | :------------------------------------- |
| `text`           | ✅  | 分析対象テキスト。最大300文字                    |
| `language`       | ✅  | テキスト言語。`ko`、`en`、`ja`のいずれか          |
| `style`          | ❌  | 感情スタイル。指定しない場合はデフォルトスタイルが使用されます    |
| `model`          | ❌  | デフォルトは`sona_speech_1`。現在はこのモデルのみ使用可能 |
| `voice_settings` | ❌  | 発話速度やピッチ調整値。結果の長さに影響を与える可能性があります  |

### 使用例
```http
POST /sandbox/predict-duration/{voice_id}
Content-Type: application/json
x-sup-api-key: [YOUR_API_KEY]
{
  "text": "This is a long-form sentence for duration prediction.",
  "language": "en",
  "style": "neutral"
}
```

### レスポンス例
```json
{
  "duration": 3.57381983
}
```
→ このテキストを生成すると約3.57秒分量の音声が作成されるという意味です。

### 活用のヒント
- **クレジットは実際に差し引かれません**（音声生成が行われないため）。
- 同じテキストを実際に呼び出す時とほぼ同様の結果を得ることができます。
- `voice_settings.speed`を調整すると長さが変わるため、固定された発話速度でテストすることをお勧めします。
